# 网络模式 -> 能力矩阵 -> 隧道映射

本文档定义 HTTP 映射隧道模式的配置约束链，确保“全局约束先于局部配置”。

## 1. 三层约束关系

1. **网络模式（全局）**：定义可用链路方向与承载边界。
2. **能力矩阵（全局推导）**：后端按网络模式推导支持/不支持能力。
3. **隧道映射（逐条配置）**：配置本端入口到对端目标，不得突破能力矩阵。

## 2. 约束落地

- 运维配置入口：隧道映射页只配置“本端入口 -> 对端目标 + 超时/体积限制 + 启用状态”。
- 校验入口：映射保存/更新时，后端依据能力矩阵做硬校验与告警返回。
- 诊断入口：`/api/startup-summary`、`/api/selfcheck`、诊断导出统一展示当前模式与校验结果。

## 3. 为什么不能把映射页做成 route/template 编辑器

- route/template 是历史模型，会弱化网络模式全局约束。
- 逐条 route/template 心智会导致运维绕过能力矩阵。
- 当前系统目标是“受全局网络模式约束的 HTTP 映射隧道系统”，而不是“api_code 模板调用系统”。

## 4. 术语规范

- 默认：本端节点、对端节点、网络模式、能力矩阵、隧道映射、本端入口、对端目标。
- 兼容：`route/api_code/template`（历史模型，deprecated 术语）。
