# GB28181 安全边界专网：端口与传输调优说明

## 推荐端口规划

- SIP 信令：`5060`
- UI 管理面：`18080`
- RTP 媒体端口段：`20000-20999`
- 节点映射端口段：`21000-21999`

这样划分的目标是：

1. 把固定入口端口（SIP/UI）与高频动态端口段（RTP/映射）彻底隔离。
2. 让 RTP 与映射端口段顺序连续，便于现场防火墙/白名单配置。
3. 让现场人员尽量不需要重新规划端口，降低错配概率。

## RTP 端口池容量计算

当前实现按“1 个 RTP 传输占用 1 个端口”估算：

- `20000-20999` 共 `1000` 个端口
- 理论上限：`1000` 路
- 推荐稳定值：预留 `20%` 余量，按 `800` 路以内规划

> 注意：这是端口池容量上限，不是链路带宽上限。现场仍需结合 CPU、NIC、边界设备策略、下游 HTTP 时延综合评估。

## INLINE 预算计算

默认公式：

```text
inline_body_budget = (udp_budget - safety_reserve - envelope_overhead - headroom) * 3 / 4
```

按默认值：

- `udp_budget = 1200`
- `safety_reserve = 220`
- `envelope_overhead = 320`
- `headroom = 1200 * 15% = 180`
- 有效线速预算：`1200 - 220 - 320 - 180 = 480`
- 推荐正文预算：`480 * 3 / 4 = 360`


## 调优原则

1. 小 JSON / 小错误页优先 INLINE，但必须在预算内。
2. 未知长度、chunked、socket.io polling、下载类响应优先 RTP。
3. RTP 长流默认启用小窗口乱序容忍，不再因 1 个 seq gap 立即失败。
4. fixed-window/resume 仍是兜底，不替代流媒体层的乱序容忍。
