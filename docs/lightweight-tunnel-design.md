# Lightweight Tunnel Gateway 设计说明

SIPTunnel 的产品定位收敛为：**轻量级隧道网关（Lightweight Tunnel Gateway）**。

## 为什么不是复杂控制平面

项目当前目标是快速稳定落地，而不是构建重型编排系统：

- 不引入复杂控制器层级与多级调度。
- 不引入重型依赖（如复杂 ORM / 多组件元数据系统）。
- 重点保证跨边界转发链路稳定、可观测、可运维。

## 轻量化原则

1. **控制面简化**：SIP 仅承载状态与控制信息。
2. **数据面稳定**：RTP 单向承载大载荷，结构固定。
3. **HTTP 执行受控**：通过隧道映射（本端入口 → 对端目标）执行。
4. **能力驱动配置**：由网络能力模式推导能力矩阵，统一约束映射。
5. **运维优先**：默认 SQLite 持久化、自动清理、内置诊断导出。

## 结果

系统对外呈现为：

网络能力模式 → 能力矩阵 → 隧道映射 → HTTP 能力

这使系统具备低学习成本、低运维负担和可预测行为，符合“轻量级隧道网关”定位。
