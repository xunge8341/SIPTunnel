# 网络能力模式与能力矩阵模型

## 1. 什么是网络能力模式

网络能力模式是本系统的**全局网络约束**，用于描述“本端节点与对端节点在当前链路下可用的业务能力边界”。

系统采用三层结构统一表达：

1. **internal enum**：后端稳定枚举，供校验、存储、接口使用。
2. **display label**：UI 展示标签，供运维人员理解。
3. **capability summary**：能力摘要（支持/不支持），供映射配置与告警联动。

## 2. 能力矩阵

能力矩阵从网络能力模式推导，不在映射层重复配置。典型维度包括：

- 请求体大小能力
- 响应体大小能力
- 流式响应能力
- 双向 HTTP 隧道能力
- 大文件承载能力

能力矩阵由后端统一生成并暴露到：

- 启动摘要
- `/api/system/status`
- `/api/tunnel/config`
- 映射校验结果

## 3. HTTP 能力约束

隧道映射（本端入口 → 对端目标）必须受能力矩阵约束：

- 超出能力的请求体/响应体限制将被拦截或告警。
- 流式响应要求与当前能力不匹配时会返回校验错误。
- 对端目标不可达与能力不匹配会在映射状态中给出阻塞原因。

## 4. 为什么 transport 由全局决定

transport（SIP/RTP 承载策略）属于链路级基础设施决策，不应在映射层逐条选择：

- 避免同一节点出现相互冲突的底层承载策略。
- 降低运维配置复杂度，减少“规则能保存但运行不一致”。
- 让能力矩阵可预测，可统一诊断与导出。

## 5. 为什么映射页不逐条选 transport

映射层关注业务映射关系：

- 本端入口
- 对端目标
- 超时与体积限制
- 启用状态

transport 是网络能力模式的一部分，由系统全局推导后自动应用。这样可保证：

- 运维输入更少
- 校验一致性更高
- 排障路径更清晰
