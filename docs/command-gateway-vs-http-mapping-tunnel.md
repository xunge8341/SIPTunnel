# 命令网关模式 vs HTTP 映射隧道模式

本文档用于消除“系统只是 api_code 模板调用系统”的误解，明确 SIPTunnel 的双模式定位与主线表达。

## 1. 术语与定位

### 1.1 命令网关模式

- 关注对象：控制命令、状态回传、任务编排。
- 主线实体：命令任务、SIP 控制消息、执行状态。
- 典型目标：实现跨域控制与状态同步。

### 1.2 HTTP 映射隧道模式（当前主线）

- 关注对象：跨安全边界 HTTP 访问映射。
- 主线实体：本端节点、对端节点、网络模式、能力矩阵、隧道映射。
- 典型目标：把本端入口请求通过 SIP/RTP 控制与承载链路映射到对端目标。

## 2. 主链路网络模型

- 接收端（SIP 下级域）：监听本端入口，收到 HTTP 请求后向发送端发起 Invite。
- 发送端（SIP 上级域）：收到 Invite 后访问对端目标 `IP:Port`，并通过 SIP/RTP 回传。
- 支持单向映射和双向映射。
- 基础承载能力来源于 GB/T 28181 视频点流链路。

## 3. 术语替换基线

当前默认术语必须使用：

- 本端节点
- 对端节点
- 网络模式
- 能力矩阵
- 隧道映射
- 本端入口
- 对端目标

以下术语统一标记为历史模型（兼容术语 / deprecated 术语）：

- `route`
- `api_code`
- `template`

## 4. 迁移与兼容原则

1. 文档/UI/注释默认使用新术语。
2. 旧术语仅在“兼容接口、历史配置、迁移工具”场景出现，并明确标注。
3. 新增功能不得继续以 `api_code` 作为产品主表达。
