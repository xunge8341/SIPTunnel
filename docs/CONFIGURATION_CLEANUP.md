# 配置项清理说明

当前配置主线以以下块为准：

- `storage`
- `sip`
- `rtp`
- `observability`
- `ui`
- `ops`

其中：

- `ui.listen_ip` / `ui.listen_port` 是管理面唯一监听配置；不再保留 `server.port` 兼容口。
- 节点、对端、会话、安全、链路能力、本地隧道映射端口范围属于**运行态业务配置**，由 `/api/node-tunnel/workspace` 与运行态数据文件统一维护。
- 本地资源与隧道映射是**两类不同数据**：
  - 本地资源：资源定义与目录发布源
  - 隧道映射：远端目录资源 + 本地监听补充信息

已废弃/不再建议使用：

- `server.*`：已移除，统一使用 `ui.*`。
- `network.*`：历史兼容块，UI 不再维护，示例配置已移除。
- `route/api_code/template`：历史模型，仅兼容迁移文档保留，不作为主配置方式。

当前 UI 已闭环的主要配置：

- 节点与级联：本端/对端、SIP/RTP、会话、安全、管理面访问控制、本地隧道映射端口范围
- 本地资源：资源编码、名称、目标 URL、方法、响应模式、目录发布
- 隧道映射：远端资源 + 本地监听端口/路径
- 系统设置：SQLite、清理、保留策略、管理面安全
- 告警与保护：限流、熔断、失败阈值
- 授权管理 / 安全事件：授权导入、机器码导出、安全事件查询
