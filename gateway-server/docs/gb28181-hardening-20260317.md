# GB/T 28181 Hardening - 2026-03-17

本轮补丁聚焦以下四类问题：

1. **Windows 运行期稳定性止血**
   - access log 改为异步入库队列，避免 HTTP 热路径同步写 SQLite。
   - SQLite 连接收敛为单连接模型，并启用 `busy_timeout`。
   - 清理阶段由 `VACUUM` 改为 `PRAGMA optimize`，降低运行期内存/文件整理冲击。

2. **SIP 标准头收敛**
   - 新增对话状态 `sipDialogState`，统一维护 `Call-ID / From / To / Contact / tag / remote target`。
   - `INVITE / ACK / INFO / REGISTER / MESSAGE / SUBSCRIBE / NOTIFY` 均使用标准头组织，不再依赖私有 `X-*` 头在发送侧传递状态。
   - `Via` 从固定 `siptunnel.local` 改为基于当前节点可达地址和监听端口动态生成。

3. **去 X 头 + 对话状态化**
   - 发送侧删除 `X-Callback-Addr / X-Device-ID / X-Mapping-ID / X-Response-Mode`。
   - 接收侧仍保留对 `X-Device-ID` 的兼容解析兜底，便于存量节点平滑升级。
   - 回调 `INFO/BYE` 等在对话内通过 `dialog state` 构造，而不是重新拼装私有头。

4. **RTP/INLINE 降级策略**
   - 当 mapping 为 `AUTO` 且请求明显属于浏览器静态资源（如 `.js/.css/.ico/.svg/.png/.woff/.json` 等）时，优先使用 `INLINE`，减少浏览器并发拉取静态资源时的 RTP 端口争用。
   - 当 RTP 端口池耗尽且模式为 `AUTO` 时，继续保留降级为 `INLINE` 的止血逻辑。
