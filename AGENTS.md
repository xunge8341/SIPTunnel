# Codex 项目说明（SIPTunnel）

## 项目定位

SIPTunnel 是跨安全边界业务交换网关：
- 控制面使用 SIP 小报文交互状态与控制信息。
- 文件面使用 RTP 单向（B -> A）承载大载荷。
- A 网通过受控 HTTP 路由模板执行本地业务。

## 工程约定

1. SIP Body 使用 JSON 承载完整业务字段，SIP Header 仅保留镜像索引字段。
2. RTP 使用固定主头 + TLV 扩展，不允许在业务代码直接拼裸字节。
3. HTTP 执行必须走 `api_code` 模板映射，禁止任意透传。
4. 签名能力从 `Signer` 接口注入，首版 HMAC-SHA256，保留国密升级位。
5. 模块边界优先，避免将幂等、补片、路由、签名耦合到单一 god object。

## 质量基线

- 每次改动至少补齐对应单元测试。
- 提交前运行 `go test ./...`。
- 协议与运行方式需同步更新 README/docs。
